# PowerChina 出差申请

这个 skill 用于处理 PowerChina 出差申请流程。

## 工作流程

1. 准备 `apply` 所需的 JSON 参数。
2. 执行 `scripts/business_trip.py`。
3. 从标准输出读取 JSON 结果。
4. 如果用户提供的出差类别不是精确枚举值，而是模糊描述，需要从允许的 `trip_type` 列表中挑选最接近的一项。
5. 如果用户提供的是模糊时间表达，需要先转换成明确的 `start_time` 和 `end_time`。
6. 在最终提交前，必须把解析后的全部参数展示给用户确认。

## 命令

```bash
python /app/nanobot/skills/powerchina-business-trip/scripts/business_trip.py --headless apply --login-json '{"usrname":"your-account","pwd":"your-password"}' --trip-json '{"trip_type":"省内公务出差","start_time":"2026-03-18 09:00:00","end_time":"2026-03-20 18:00:00","destination":"北京","trip_reason":"项目现场沟通"}'
```

## 输入格式

出差申请 JSON：

```json
{
  "trip_type": "省内公务出差",
  "start_time": "2026-03-18 09:00:00",
  "end_time": "2026-03-20 18:00:00",
  "destination": "北京",
  "trip_reason": "项目现场沟通"
}
```

必填字段：

- `usrname`
- `pwd`
- `trip_type`
- `start_time`
- `end_time`
- `destination`

可选字段：

- `trip_reason`

允许的 `trip_type` 值：

- `省内公务出差`
- `外地出差`
- `国外公务出差`
- `本地出差`
- `省外（发达）公务出差`
- `国外工地出差`
- `国内工地出差`
- `省外（一般）公务出差`

## 时间规则

公司的标准上班时段是 `09:00-17:00`。

如果用户时间描述模糊，按下面规则解析：

- 用户说“全天”，按所述日期 `09:00:00-17:00:00` 处理。
- 用户说“半天”，如果明确说“上午”，按 `09:00:00-12:00:00` 处理。
- 用户说“半天”，如果明确说“下午”，按 `12:00:00-17:00:00` 处理。
- 用户只给日期、没有给具体时间时，默认按全天处理，如果需要则向用户确认。
- 用户说“今天下午”“明天上午”“后天全天”这类相对时间时，要先换算成明确日期，再生成完整时间。
- 用户说“从某天到某天”这种多天出差时，开始时间和结束时间要能完整包住中间所有工作日的上班打卡时段。
- 多天出差默认应覆盖首日 `09:00:00` 到末日 `17:00:00`，除非用户明确要求首日只上午/下午、末日只上午/下午。
- 如果用户给出的时间范围不能覆盖应有的上班时段，需要先纠正或向用户确认，再提交。

## 处理规则

- `trip_type` 必须视为枚举值，不能自行创造新类别。
- 如果用户给出的类别是模糊表达，要明确映射到最接近的允许值。
- `start_time` 和 `end_time` 在提交前必须是明确的完整时间字符串。
- 在真正提交前，要把最终解析出的完整参数回显给用户确认。
- 建议在确认时至少回显：`trip_type`、`start_time`、`end_time`、`destination`、`trip_reason`。
- 如果缺少字段，直接返回缺少哪些字段，不要猜测。
